Install React Native
---

# ✅ Step 1: Install Homebrew (macOS package manager)

**Why?** Homebrew makes it easy to install packages and dependencies.

* Open Terminal and run:

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

* After installation, run:

```bash
brew doctor
```

* **Homebrew website**: [https://brew.sh/](https://brew.sh/)

---

# ✅ Step 2: Install Node.js and Watchman

**Why?** Node.js is required to run React Native CLI and npm/yarn. Watchman improves file watching.

* Install using Homebrew:

```bash
brew install node
brew install watchman
```

* **Node.js website**: [https://nodejs.org/](https://nodejs.org/)
* **Watchman repo**: [https://facebook.github.io/watchman/](https://facebook.github.io/watchman/)

---

# ✅ Step 3: Install JDK (Java Development Kit)

**Why?** Required for building Android apps.

* Recommended: **Zulu JDK 17** (supported by React Native and Android Studio).

* Download and install from here:
  [https://www.azul.com/downloads/?version=java-17-lts\&os=macos\&architecture=arm-64-bit\&package=jdk](https://www.azul.com/downloads/?version=java-17-lts&os=macos&architecture=arm-64-bit&package=jdk)

---

# ✅ Step 4: Install Android Studio (for Android development)

**Why?** Provides Android SDK and emulator.

* Download: [https://developer.android.com/studio](https://developer.android.com/studio)
* During installation, select these components:

  * Android SDK
  * Android SDK Platform
  * Android Virtual Device (AVD)

**After install:**

1. Open Android Studio → Preferences → Appearance & Behavior → System Settings → Android SDK.
2. Install:

   * Android SDK Platform 34 (or latest stable)
   * Android SDK Build-Tools
   * Intel x86 Emulator Accelerator (HAXM) or ARM-based emulators (depending on your Mac).

---

# ✅ Step 5: Set up Android environment variables

Add these lines to your `~/.zprofile` or `~/.bash_profile` (depends on your shell):

```bash
export ANDROID_HOME=$HOME/Library/Android/sdk
export PATH=$PATH:$ANDROID_HOME/emulator
export PATH=$PATH:$ANDROID_HOME/tools
export PATH=$PATH:$ANDROID_HOME/tools/bin
export PATH=$PATH:$ANDROID_HOME/platform-tools
```

Then run:

```bash
source ~/.zprofile
```

---

# ✅ Step 6: Install Xcode (for iOS development)

**Why?** Required to build and run iOS apps.

* Install from the App Store: [https://apps.apple.com/us/app/xcode/id497799835](https://apps.apple.com/us/app/xcode/id497799835)
* After installing, run:

```bash
sudo xcode-select --switch /Applications/Xcode.app/Contents/Developer
sudo xcodebuild -runFirstLaunch
```

---

# ✅ Step 7: Install CocoaPods

**Why?** Manages iOS native dependencies.

* Install using Homebrew:

```bash
sudo gem install cocoapods
```

✅ Alternative (recommended if you use Homebrew):

brew install cocoapods

* **CocoaPods website**: [https://cocoapods.org/](https://cocoapods.org/)

---

# ✅ Step 8: Create a new React Native project

**If you want to use the React Native CLI (recommended for full flexibility):**

```bash
npx react-native init MyApp
cd MyApp
npx react-native run-ios     # for iOS
npx react-native run-android # for Android
```

OR 

npx @react-native-community/cli init MyApp


**If you prefer Expo (easier, but limited native code customization):**

```bash
npm install -g expo-cli
expo init MyApp
cd MyApp
expo start
```

* **Expo website**: [https://expo.dev/](https://expo.dev/)

---

# ✅ Step 9: Configure simulators/emulators

* For iOS: Open Xcode → Window → Devices and Simulators → Add a new simulator.
* For Android: Open Android Studio → Tools → Device Manager → Create Virtual Device.

---

# ✅ Step 10: Test and start coding

Run your app on the simulator or physical device and start editing `App.js`.

ℹ️ Extra Tips

If you still see errors, you may need to install CocoaPods dependencies:

cd ios
pod install
cd ..


Then rerun:

npx react-native run-ios

Yes, for live deployment you should change the API base URL.

Update this file:

config.ts
Right now it uses local dev hosts:

iOS: http://localhost:8000
Android emulator: http://10.0.2.2:8000
For production, set your real backend domain, for example:

export const API_BASE_URL = 'https://api.yourdomain.com';
export const API_V1_BASE_URL = `${API_BASE_URL}/api/v1`;
Also ensure on server:

HTTPS is enabled (recommended/required for live apps).
Sanctum/API auth works on your live domain.
CORS allows your app requests.
